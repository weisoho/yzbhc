# 悬浮备注窗口使用说明

## 功能概述

悬浮备注窗口是一个为开发团队设计的协作工具，允许开发人员在各个页面上添加、编辑和保存备注信息。这些备注会与特定页面关联，并在页面切换时自动同步显示。

## 主要特性

### 1. 基础功能
- **可拖拽移动**：拖动窗口标题栏可以移动窗口位置
- **可调整大小**：拖动窗口右下角可以调整窗口大小
- **文本输入区域**：支持多行文本输入，类似记事本应用
- **窗口控制**：支持最小化、最大化、关闭操作

### 2. 数据管理
- **自动保存**：输入内容2秒后自动保存
- **本地存储**：使用localStorage持久化存储，刷新页面不丢失
- **页面关联**：每个页面的备注独立存储，互不干扰
- **导入导出**：支持备注数据的JSON格式导入导出

### 3. 页面同步
- **自动切换**：切换页面时自动显示对应页面的备注
- **实时保存**：切换页面时自动保存当前编辑内容
- **上下文感知**：根据当前页面路径加载对应备注

### 4. 用户体验
- **默认位置**：窗口默认显示在屏幕右下角
- **响应式设计**：适配不同屏幕尺寸
- **动画效果**：窗口移动、显示/隐藏有平滑动画
- **视觉反馈**：拖拽、调整大小时有视觉提示

## 使用方法

### 基本操作
1. **显示/隐藏窗口**：点击右下角的蓝色笔记图标
2. **移动窗口**：拖动窗口顶部的标题栏
3. **调整大小**：拖动窗口右下角的调整手柄
4. **编辑备注**：点击编辑按钮进入编辑模式
5. **保存备注**：点击保存按钮或等待自动保存

### 窗口控制按钮
- **编辑按钮**：切换编辑模式（蓝色表示编辑中）
- **保存按钮**：手动保存当前备注
- **最小化按钮**：将窗口最小化为标题栏
- **最大化按钮**：将窗口最大化到全屏
- **关闭按钮**：隐藏悬浮窗口

### 数据操作
- **清空**：清空当前页面的备注内容
- **导出**：将所有备注导出为JSON文件
- **导入**：从JSON文件导入备注数据
- **设置**：查看数据统计和使用说明
- **关于**：查看功能特性和技术信息

## 技术实现

### 架构设计
```
App.jsx
├── NoteProvider (上下文管理)
├── GlobalNoteWindow (全局控制)
└── FloatingNoteWindow (主窗口)
```

### 数据存储
- **存储键**：`floating_notes_data`
- **数据结构**：按页面路径组织的JSON对象
- **存储位置**：浏览器localStorage
- **数据格式**：
  ```json
  {
    "/page-path": {
      "id": "unique-id",
      "pagePath": "/page-path",
      "pageTitle": "页面标题",
      "content": "备注内容",
      "lastModified": "ISO时间戳",
      "createdBy": "创建者"
    }
  }
  ```

### 页面关联机制
1. 使用React Router的`useLocation`获取当前页面路径
2. 通过`NoteContext`管理页面切换时的数据同步
3. 每个页面路径对应独立的存储键
4. 切换页面时自动加载对应备注

## 响应式设计

### 桌面端
- 默认窗口尺寸：400x300像素
- 最小尺寸：300x200像素
- 最大尺寸：800x600像素
- 拖拽区域：整个标题栏
- 调整手柄：右下角16x16像素区域

### 移动端
- 窗口宽度：95%视口宽度
- 按钮布局：垂直排列
- 触摸优化：增大点击区域
- 手势支持：拖拽和调整大小

### 屏幕适配
- 小屏幕（<768px）：简化布局，垂直排列按钮
- 中等屏幕（768px-1024px）：标准布局
- 大屏幕（>1024px）：优化间距和字体大小

## 协作功能

### 团队使用
1. **上下文共享**：开发人员可以在特定页面留下开发说明
2. **问题记录**：记录页面特定的bug或注意事项
3. **配置信息**：保存测试数据或配置信息
4. **进度跟踪**：记录开发进度和待办事项

### 数据迁移
1. **导出数据**：使用导出功能备份所有备注
2. **导入数据**：在新环境中导入备注数据
3. **数据合并**：导入时自动合并，新数据覆盖旧数据
4. **格式兼容**：使用标准JSON格式，便于处理

## 故障排除

### 常见问题
1. **窗口无法拖拽**：检查是否处于最小化或最大化状态
2. **备注未保存**：检查页面路径是否正确，内容是否为空
3. **导入失败**：检查JSON文件格式是否正确
4. **样式异常**：检查浏览器是否支持CSS Grid和Flexbox

### 数据恢复
1. **本地备份**：定期导出备注数据作为备份
2. **数据清理**：使用浏览器开发者工具清理localStorage
3. **重置状态**：刷新页面重新加载组件状态
