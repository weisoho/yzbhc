3.此时就可以在创建商品目录时，选择供应商、选择并绑定注册证，之后继续手动录入其他商品信息，保存明细，提交商品信息，系统生成商品目录唯一码（商品编码），该商品目录在系统内可用，完成商品目录建立流程。（创建的商品明细呈现在字典维护/物资字典内）

树状图层级关系：

注意：同一个商品明细记录不可以重复绑定不同供应商/注册证，具有唯一性。



（二）院区建立/科室建立/账户建立
同上，院区为一级目录，院区下科室为二级目录，这里先从单一院区开始：
默认院区为初始院区，对应需求当中的查看所有院区库存、目录等操作的需求（如果不需要也能实现则忽略这一条）

（默认为空院区）新建院区（输入文件夹名称）→→→新建科室（在院区文件夹下面再建几个文件夹，输入你想要的名称，以后它就叫“科室”了）→→→完成科室和院区创建

创建新账号（新建一个名叫“张三”的文件）→→→选择科室（你要把它放哪个文件夹下面，它就属于哪个科室的，拥有可操作那个科室的权限。如何划分不同人员的权限：将操作权限写在“张三”的文件内。要同时拥有不同科室的操作权限，只需要把“张三”文件拖到更上一级的目录即可，或者在每一个“科室”文件夹内都放一个“张三”的文件）→→→完成账号创建

角色模板：创建角色时，选择所属科室后，要给他什么访问权限，限定他可以操作的内容，提前设定好模板之后，就可以一键添加权限。（包括单一权限或页面的访问权限，需要手动详细设定/或者同时勾选数个角色模板）

订单状态分为（待提交、待审核/已驳回、待收货、待入库、已完成）

采购单据完成提交步骤进入（待审核）状态时，就会生成一个对应的【采购单号】


1、采购

由科室发起采购申请（进入采购申请界面、获取全院商品目录，筛选搜索条件并设定需要采购的商品，录入需要采购的数量，提交单据）→采购单据进入科室采购审核（由科室负责人审核或指定权限人），可进行通过、修改采购订单、驳回三种操作，通过采购申请后，采购单据进入到【采购收货】页面

2.采购收货


对已经完成采购审核的订单进行确认收货操作，在这里需要核对[实际到货的数量]填入并完成收货，订单完成收货后，进入【验收入库】详细商品信息录入流程
到货数量与采购数量一致，确认收货即可；
实际到货数量因为各种原因不足采购数量时，在备注栏手动填写原因，提交后将按照[实际填写数量]提交订单，不足采购数量的部分，将推送到货数量不足的采购申请记录到【异常订单管理】，原采购订单收货状态改为（待入库）

3.【验收入库】
完成收货的订单，将会进入【验收入库】页面。[查单据]界面显示所有已完成收货的采购单，点击对应的采购单之后，就可以录入批号效期等详细信息。

录入信息时，默认只显示数条不重复、唯一商品编码的待录入信息目录。
当实际货物出现可以允许的同一商品不同效期、批号的情况，先修改【入库数量】对应批号货物的数量，双击该条明细新增该商品的新空明细行（就是没人录入批号效期的空白行，左边显示剩余没有入库的采购数量）。
最终实现：采购10箱注射器，5箱、3箱、2箱都是不同批号效期，但是总数量一致。（交单据时不能超出采购数量）


4、消耗出库
有对应操作权限的人即可进行消耗出库操作。撤销消耗由对应科室负责人负责管理

5、结算
按照当月入库数据/消耗数据生成结算数据，并按照供应商合并各个科室数据
结算数据由院区财务部门管理。

6、物资调拨
由科室负责人发起调拨申请，选择急需调拨到耗材，然后可以查看拥有对应耗材的全院库存，选择科室并提交调拨申请。

方案一：由被调拨科室负责人审核，是否同意或拒绝
方案二：由上级管理部门处理调拨申请，并查看全院科室库存，选择调拨科室调拨物资。

物资被调拨后，原始入库数据不变，当月核算成本依旧算在初始入库科室。

### 耗材管理系统核心逻辑梳理（SPD模式）

#### 1. 核心数据实体与关系
系统基于以下核心数据实体构建，实体间关系通过外键关联。

| 实体 | 关键属性 | 状态字段 | 说明 |
|------|----------|----------|------|
| **供应商（Supplier）** | 供应商ID（主键）、名称（唯一）、类型（经营企业/生产企业/医疗机构）、联系人、电话等 | status（0-不可用，1-可用） | 新建时默认为不可用，上传经营许可证并保存后自动变为可用。 |
| **注册证（Certificate）** | 注册证ID、注册证名称、编号、有效期、附件路径等 | status（0-不可用，1-可用） | 新建时需关联供应商，上传附件保存后变为可用。同一注册证名称可被多个供应商引用（即允许多个供应商拥有相同名称的注册证记录）。 |
| **商品目录（Product）** | 商品编码（唯一，按规则生成）、商品名称、规格型号、单位、生产厂家、供应商ID、注册证ID、价格等 | status（0-不可用，1-可用） | 新建时必须选择已激活的供应商和注册证，保存后生成唯一商品编码，状态默认为可用。一个商品只能属于一个供应商下的一个注册证（即供应商ID+注册证ID+商品信息组合唯一）。 |
| **院区（Campus）** | 院区ID、名称 | - | 支持多院区扩展，默认存在一个初始院区。 |
| **科室（Department）** | 科室ID、名称、所属院区ID | - | 属于某个院区，一个院区下可建多个科室。 |
| **用户（User）** | 用户ID、登录名、密码、姓名、所属科室列表（多对多） | - | 用户可被分配至多个科室（通过中间表），权限通过角色模板授予。 |
| **角色（Role）** | 角色ID、名称、权限集合（JSON或权限ID列表） | - | 预定义的权限模板，可分配给用户，并可限定生效的科室范围（全局或指定科室）。 |
| **采购订单（PurchaseOrder）** | 订单号（唯一，自动生成）、申请科室ID、供应商ID、订单状态、创建时间、审核时间等 | status（待提交、待审核、已驳回、待收货、待入库、已完成） | 由科室发起采购申请，审核通过后进入待收货状态。 |
| **采购订单明细（PurchaseOrderItem）** | 明细ID、订单ID、商品编码、采购数量、实际收货数量、单价等 | - | 关联商品。 |
| **收货单（ReceivingNote）** | 收货单号、采购订单号、收货人、收货时间、备注等 | status（待入库、已完成） | 采购收货后生成，对应采购订单的部分或全部到货。 |
| **入库单（StockIn）** | 入库单号、收货单号、入库科室ID、入库时间、操作人等 | status（草稿、已完成） | 对收货单进行批号/效期录入，生成库存记录。 |
| **入库明细（StockInItem）** | 明细ID、入库单号、商品编码、批号、生产日期、有效期、入库数量、采购价等 | - | 记录每个批次的详细信息。 |
| **库存记录（Inventory）** | 库存ID、商品编码、科室ID、批号、有效期、库存数量 | - | 实时库存，每次入库/出库/调拨更新。 |
| **出库单（StockOut）** | 出库单号、出库科室、出库类型（消耗/调拨）、出库时间、操作人等 | status（草稿、已完成） | 消耗出库或调拨出库的记录。 |
| **调拨申请（TransferRequest）** | 申请单号、发起科室、目标科室、商品编码、申请数量、状态、审核人等 | status（待审核、已同意、已拒绝、已完成） | 科室间调拨的申请流程。 |
| **异常订单（ExceptionOrder）** | 异常单号、原采购订单号、商品编码、短缺数量、原因、处理状态等 | status（待处理、已处理） | 采购收货时数量不足的记录。 |
| **结算单（Settlement）** | 结算单号、结算周期、供应商ID、总金额、状态等 | status（草稿、已确认、已付款） | 按月生成，汇总各科室入库或消耗数据。 |

#### 2. 状态机与业务规则
##### 2.1 实体状态流转
- **供应商**：`新建(不可用)` → `上传经营许可证并保存` → `可用`
- **注册证**：`新建(不可用)` → `上传附件并保存` → `可用`（注册证状态独立于供应商，但其所属供应商必须为可用）
- **商品目录**：`新建(可用)`，可手动禁用/启用

##### 2.2 采购订单状态机
```
待提交（科室草稿） → 待审核（提交后） → 已驳回（审核不通过，可重新编辑）  
待审核 → 待收货（审核通过）  
待收货 → 待入库（收货后，但需验收入库）  
待入库 → 已完成（验收入库完成）
```
- 采购订单提交时生成唯一采购单号。

##### 2.3 业务约束
- 供应商名称全局唯一。
- 商品编码生成规则：建议采用前缀+年月日+流水号（如`SP-20250310-0001`），按新增顺序分配。
- 入库数量总和不能超过对应采购订单明细的采购数量（允许分批次入库）。
- 同一次入库中，同一商品允许多个不同批号/效期，通过入库明细行分别录入。
- 调拨不改变库存成本归属（原始入库科室的成本不变，调拨后接收科室的库存成本仍按原始采购价核算，但实际财务结算可按调拨协议处理，此处按原文“核算成本依旧算在初始入库科室”）。

#### 3. 核心业务流程详解
##### 3.1 商品目录建立流程
1. **新建供应商**：录入供应商信息，保存后供应商状态为不可用。
2. **上传经营许可证**：进入供应商详情页，上传经营许可证附件并保存，系统自动将供应商状态更新为可用。
3. **为供应商添加注册证**：在供应商下新建注册证记录，填写注册证信息并上传附件，保存后注册证状态变为可用（注册证可被多个供应商复用，即不同供应商可以录入相同的注册证名称，但视为不同记录）。
4. **新建商品**：进入商品目录管理，点击“新增”，选择已可用的供应商，然后选择该供应商下已可用的注册证，填写商品其他信息（名称、规格、单位等），保存后系统自动生成唯一商品编码，商品状态为可用。商品信息存入物资字典。

##### 3.2 院区、科室、用户与权限配置
1. **院区建立**：新建院区记录（如“总院”），作为组织根节点。
2. **科室建立**：在院区下新建科室记录（如“手术室”）。
3. **用户创建**：新建用户账号，填写基本信息，并分配所属科室（可多选）。用户权限不直接定义，而是通过角色模板授予。
4. **角色模板**：创建角色（如“科室护士长”），定义该角色可访问的菜单、按钮、数据范围（如只能操作本科室数据或全院数据）。角色可限定生效科室：若角色为“全院管理员”，则生效科室为“所有科室”；若角色为“科室护士长”，则需在给用户分配该角色时指定具体科室（即用户+角色+科室的关联）。用户最终权限为所有角色分配的并集。

##### 3.3 采购至结算全流程
1. **采购申请**：
   - 科室用户进入采购申请界面，从全院商品目录中选择商品，填写采购数量，生成采购申请单（状态：待提交）。
   - 提交后，申请单进入该科室指定审核人（或上级）的待办列表，状态变为待审核。
2. **采购审核**：
   - 审核人可查看申请单，执行通过、修改订单或驳回操作。
   - 若通过，订单状态更新为待收货，并进入【采购收货】列表；若驳回，状态变为已驳回，申请科室可修改后重新提交。
3. **采购收货**：
   - 仓库或指定人员进入【采购收货】页面，查看待收货订单。
   - 点击订单，核对实物，填写实际到货数量。若到货数量与采购数量一致，直接确认收货，生成收货单（状态：待入库）；若到货数量不足，填写实际数量及原因，确认后生成收货单（待入库），同时系统自动创建异常订单记录，记录短缺商品及数量，原采购订单状态更新为待入库（实际数量部分待入库，短缺部分后续补货或取消）。
4. **验收入库**：
   - 进入【验收入库】页面，查看所有待入库的收货单。
   - 点击收货单，进入入库操作界面：系统默认按商品编码分组显示待入库商品（每种商品一行），需录入每个商品的批号、生产日期、有效期、入库数量（默认等于待入库数量）。
   - 若同一商品有多个批号，可双击该行新增空明细行，分别录入不同批号的数量（总和不超过待入库数量）。
   - 录入完成后提交，生成入库单，库存相应增加，收货单状态变为已完成，采购订单状态变为已完成（若全部收货完成；若部分收货且有异常单，则采购订单状态仍为待入库，直至异常处理完毕）。
5. **消耗出库**：
   - 科室用户发起消耗出库，选择商品、数量，确认后生成出库单，减少对应科室库存。撤销消耗需科室负责人权限，可作废出库单回退库存。
6. **结算**：
   - 每月末，系统根据入库单数据（或消耗数据，按配置）按供应商汇总各科室的耗材金额，生成结算单草案。
   - 财务部门审核结算单，确认无误后标记为已确认，进行付款流程。
7. **物资调拨**：
   - 发起科室创建调拨申请，选择所需商品及数量，系统自动显示全院有库存的科室列表（排除发起科室），选择目标调出科室后提交。
   - 调拨申请进入审核流程，可配置为需目标科室负责人同意，或由上级管理部门统一处理。
   - 审核通过后，系统生成调拨出库单（从调出科室出库）和调拨入库单（在调入科室入库），库存转移。成本核算时，调入科室的入库成本仍记录为原始采购成本，便于统计各科室实际消耗成本（原始入库科室不变）。

#### 4. 关键设计点说明
- **状态控制**：所有关键实体（供应商、注册证、商品）均设有状态字段，用于控制业务操作的前置条件（如不可用供应商不可用于新建商品）。
- **唯一性约束**：供应商名称唯一；商品编码唯一；商品与供应商+注册证的组合唯一。
- **灵活权限**：通过用户-角色-科室的三元组关系，实现精细化权限控制，支持跨科室权限（如用户可同时拥有多个科室的操作权）。
- **采购收货容错**：支持部分到货，自动生成异常订单，便于后续跟踪处理，不阻塞正常入库。
- **批号效期管理**：入库时必须录入批号和效期，支持同一商品多批次入库，保证库存追溯。
- **成本核算独立性**：调拨不影响原始入库成本，便于按科室核算实际消耗（尽管实物转移，但成本仍归属最初入库科室，符合医院精细化管理需求）。
